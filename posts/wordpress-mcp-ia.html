WordPress se conecta a la IA: el adaptador MCP explicado sin tecnicismos

Por El Equipo

Imagina que tienes un asistente de inteligencia artificial —como Claude o Copilot— y le preguntas: «¿Cuántas entradas tiene mi web de WordPress?». Hasta hace poco, el asistente no tenía forma de saberlo: simplemente no tenía acceso a tu sitio. Con el nuevo adaptador MCP de WordPress, eso está empezando a cambiar. WordPress puede ahora «hablar» directamente con herramientas de IA, de forma segura y controlada.

¿Qué es el MCP y por qué importa?

MCP son las siglas de Model Context Protocol, un estándar abierto que permite a los asistentes de IA recibir información de servicios externos. Funciona como un traductor universal: en lugar de que cada herramienta de IA tenga que aprender a comunicarse con cada plataforma por separado, el MCP establece un lenguaje común. Por su parte, WordPress 6.9 introdujo la Abilities API (API de Capacidades), que es una forma ordenada de declarar qué cosas puede hacer un sitio WordPress: obtener información del sitio, consultar datos de usuario, ejecutar diagnósticos, etc. El adaptador MCP actúa de puente entre ambos mundos: convierte esas «capacidades» de WordPress en herramientas que una IA puede descubrir y utilizar directamente.

«Si tu código ya registra capacidades, estás a un paso de dejar que un agente de IA las use.» — Jonathan Bossenger, WordPress Developer Blog

En la práctica, esto significa que puedes abrir tu editor de código favorito (Cursor, VS Code) o una aplicación como Claude Desktop, hacer una pregunta sobre tu WordPress y el asistente consultará el sitio, obtendrá la respuesta y te la mostrará, todo sin que tú hayas escrito una sola línea de código adicional. Es el primer paso hacia una integración real entre los gestores de contenido y los agentes de inteligencia artificial: no la IA generando texto a ciegas, sino la IA con acceso real y estructurado a los datos de tu sitio.
